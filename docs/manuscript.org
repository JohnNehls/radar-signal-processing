* SAR
Using the Range-Doppler Algorithm (RDA), specifically the azimuth compression step.

The full RDA pipeline is:
  1. Range compression (matched filter along fast-time)
  2. Range cell migration correction (RCMC) — skipped here since it's negligible for the geometry/bandwidth used
  3. Azimuth compression — matched filter along slow-time using the hyperbolic phase history, which is what azimuth_matched_filter implements

The per-range-bin FFT correlation against  $$h_{ref} = exp\left(-j  \frac{4\pi}{\lambda}  \sqrt{R_0^2 + x^2}\right)$$ is the textbook azimuth focusing step of the RDA. It's sometimes called the "azimuth matched filter"   or "azimuth compression" interchangeably in the SAR literature (e.g., Cumming & Wong, Digital Processing of Synthetic Aperture Radar Data).
